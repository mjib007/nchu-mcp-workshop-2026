Why · 為什麼選這個教材

三件市面上少見的事

多數 MCP 教學要不是只給 spec / 規格,要不就是「先講術語再說故事」。我們反過來。

01

為非資工教師寫

所有段落都假設聽眾是熟悉 LLM 使用但不寫程式的大專教師。比喻先行、術語後補,N×M → N+M 用視覺講而不是公式講。

02

完整的故事線

不是斷裂的章節集合 — Segment 1–5 是「為什麼→怎麼運作→如何拿來用→自己做做看→上線實務」的閉環,每段都鋪陳下一段。

03

兩個 60-min 內可跑的子專案

不只給概念,給可跑的 code。mini-project 5 步驟 setup + L1–L3 三關 Lab,讓「我自己領域的助理」在課堂上發生。

SEG · 01

Why MCP

50 min v2 完成

從 LLM 擅長的事與天花板出發,介紹三種策略——RAG、Tool Use、MCP——並深入比較它們的差異與適用場景。最後引入興大 AI 學伴 33 工具案例。

SEG · 02

How MCP Works

50 min v3 完成

從 function calling 的底層機制談起(LLM 只吐字串,真的執行的是 harness),再用一個具體查詢「幫我查最近圖書館有什麼新書」串連 MCP 的兩階段握手與工具呼叫。

  • 01 Function Calling 怎麼運作:LLM 只吐字串,harness 真的執行;subprocess.run 那道門;安全警告
  • 02 從一個查詢開始:場景 → spawn → 兩階段握手 → 工具呼叫
  • 03 Tool 註冊與描述:JSON Schema、description 對 LLM 的重要性
  • 04 Client 整合機制:工具清單如何餵給 LLM

SEG · 05

實務考量

10 min 完成

四大支柱(從 mini-project 往外擴):

  • Scale 質變:3 tools → 239 tools 的五個質變
  • Tool-calling 品質:docstring 設計、邊界防禦、錯誤當資料
  • 模型選擇:task-complexity gradient、benchmark 佐證
  • 成本:10 人 workshop 真實帳、prompt caching、Anthropic Console 護欄

10 分鐘塞不下四支柱,講師現場依聽眾屬性擇一深入 ~7 min,其餘各一句帶過。md 保留完整四節供課後閱讀。
